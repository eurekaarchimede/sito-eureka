setup richiesto

questa piattaforma usa supabase come backend (free tier). setup di ~5 minuti, una volta sola.

  1. vai su supabase.com → start your project → registrati con github o email.
  2. crea un nuovo progetto. nome a piacere (es. eureka-mozioni). regione: europe west (frankfurt). password DB: salvala da qualche parte.
  3. aspetta 1–2 minuti il provisioning.
  4. nella sidebar del progetto: SQL Editor → new query. apri il file mozioni-setup.sql di questa cartella, copia tutto, incolla, premi run.
  5. Authentication → URL Configuration: imposta Site URL all'url dove servirai questa pagina (es. http://localhost:8000 o l'url di netlify/vercel/github pages). aggiungi lo stesso valore in Redirect URLs.
  6. Project Settings → API: copia Project URL e anon public key.
  7. apri mozioni.html in editor, sostituisci i due placeholder all'inizio dello <script type="module">: 
    const SUPABASE_URL = 'https://xxx.supabase.co';
const SUPABASE_ANON_KEY = 'eyJhbGc...';
  8. ricarica questa pagina e fai login con la tua email (warmamiel@gmail.com — già configurata come admin nello script SQL).

la anon key è pubblica per design — RLS protegge i dati. non committare la service_role key.
